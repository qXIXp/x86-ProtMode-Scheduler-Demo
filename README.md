# x86 Protected Mode Multitasking Scheduler
基于 x86 保护模式的四任务优先数时间片调度演示系统。以 COM 程序格式在 FreeDOS 下引导，实现 Ring3 至 Ring0 的硬件级上下文切换与分页管理。

本项目是一个轻量级的 x86 保护模式内核演示程序。项目通过在 FreeDOS 环境下执行 COM 程序引导进入 32 位保护模式，并演示了基于硬件中断（IRQ0）的多任务优先数时间片调度算法、分页机制（CR3）以及特权级切换（Ring 3 与 Ring 0）。

## 核心特性

* **运行环境切换**：通过实模式下的 INT 15h (E820) 探测物理内存，随后平滑过渡至 32 位保护模式，并在运行结束后可安全返回 DOS 实模式。
* **内存分页隔离**：建立平坦内存模型，并为 4 个任务分别构建独立的页目录（PD）与页表（PT），物理地址从 2MB 处起始，实现恒等映射。
* **硬件级任务切换**：利用局部描述符表（LDT）和任务状态段（TSS）实现不同特权级（Ring 3 与 Ring 0）之间的硬件级上下文切换。
* **中断处理机制**：初始化 8259A PIC 与 8253A 定时器，接管 IRQ0 时钟中断作为调度驱动，并实现 INT 80h 软中断演示。

## 调度算法详解（优先数机制）

系统通过 8253A 定时器产生 20Hz 的时钟中断（每 Tick = 50ms），对 4 个并发任务进行抢占式调度。

* **初始优先级/时间片配额**：
  * Task 0 (`VERY`): 16 (单次运行 800ms)
  * Task 1 (`LOVE`): 10 (单次运行 500ms)
  * Task 2 (`HUST`): 8 (单次运行 400ms)
  * Task 3 (`MRSU`): 6 (单次运行 300ms)

* **调度逻辑**：
  1. 每次 IRQ0 触发时，当前任务的剩余时间片递减。
  2. 若时间片未归零，则直接通过 `iretd` 返回原任务继续执行。
  3. 若时间片归零，扫描所有任务，选取当前剩余时间片最大（即优先级最高）的就绪任务。
  4. 若所有任务的时间片均耗尽，则按照初始配额重新填满各任务的时间片池，再执行步骤 3。
  5. 切换目标任务时，重置内核栈以防止栈泄漏，加载新的 LDT 与 CR3，最终进入 Ring 3 用户态执行。

## 文件结构

* `main.asm`：主程序入口，包含 16 位实模式初始化代码、32 位保护模式内核逻辑、GDT/IDT 定义以及 IRQ0 中断处理程序。
* `defs.inc`：核心宏定义文件。包含段描述符、门描述符的属性常量，以及用于批量生成任务结构（LDT、TSS、页表、内核/用户栈）的汇编宏。
* `utils.inc`：工具函数库。包含可编程控制器（8259A/8253A）的初始化例程及其他底层操作辅助函数。

## 编译与运行

### 依赖工具
* **NASM**：用于编译汇编源码。
* **Bochs**：用于运行和调试 x86 系统的虚拟机。
* **FreeDOS**：作为宿主操作系统环境加载 COM 文件。

### 构建指令
使用 NASM 将源码编译为 DOS COM 可执行文件：

```bash
nasm main.asm -o kernel.com

```

### 运行步骤

1. 将生成的 `kernel.com` 放入包含 FreeDOS 的虚拟机磁盘镜像中。
2. 启动 Bochs 虚拟机并进入 FreeDOS 命令行。
3. 执行 `kernel.com`，系统将输出内存布局并开始多任务调度演示（各任务在屏幕第 20 行以不同颜色打印其标识符）。
